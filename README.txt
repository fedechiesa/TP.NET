8) Que es el IL?

	Las siglas IL significan Intermediate Lenguaje (Lenguaje Intermedio), este es utilizado
el contexto de la compilacion de codigo.
	 Es un lenguaje de programación de nivel intermedio diseñado para ser utilizado como una 
representación intermedia entre el código fuente y el código de máquina específico de una
 plataforma.
	El objetivo principal del IL es permitir la portabilidad del código fuente, ya que el
 código IL puede ser compilado y ejecutado en diferentes plataformas de destino.


9) Explique con sus palabras VES, CTS y CLS.
	
	VES (Virtual Execution System, Sistema de Ejecución Virtual): El VES es el entorno de
ejecución en el que se ejecutan las aplicaciones desarrolladas para el framework .NET.
	CTS (Common Type System, Sistema de Tipos Común) define cómo se representan y se 
acceden a los tipos de datos en .NET.
	CLS (Common Language Specification, Especificación de Lenguaje Común) establece las 
reglas para garantizar la interoperabilidad entre los lenguajes de programación en el framework .NET. 


10) Explique con sus palabras que es un Assembly, el early y el late Binding.

	Assembly es como un contenedor que guarda todas las partes y piezas del código de un programa, 
ya sea un programa ejecutable o una biblioteca de clases. Es el archivo que contiene todo lo necesario
para que el programa funcione correctamente.
	Early Binding, cuando lo usas el compilador revisa el código antes de ejecutarlo y se asegura 
de que todas las referencias a variables, funciones o métodos estén correctamente definidas y 
disponibles en tiempo de compilación.
	Late Binding es al contrario del Early, en vez de resolver referencias y conexiones en tiempo
de compilacion, este permite que se resuelvan en tiempo de ejecucion. Permite una mayor flexibilidad
y adaptibilidad del codigo.


11) Que es GIT? Es centralizado o distribuido? Ejemplifique al menos 3 comandos de git y explique su 
funcionamiento.

	Git es un sistema de control de versiones ampliamente utilizado en el desarrollo de software. 
Permite rastrear los cambios realizados en los archivos de un proyecto a lo largo del tiempo, 
facilitando la colaboración entre desarrolladores y la gestión de versiones del software.
	Git es un sistema distribuido o descentralizado ya que permite realizar cambios sin necesidad de 
estar conectados a un servidor central.
	Permite que cada usuario/desarrollador trabaje de manera autonoma en una copia del "proyecto"
antes de aplicar los cambios en el "main".

	Comandos de GIT:
	git status : es un comando en Git que muestra el estado actual del repositorio y los archivos en él
que han sido modificados y aun no han sido agregados por ejemplo.
	git add : comando de git utilizado para agregar los cambios realizados.
	git branch : es utilizado para administrar y trabajar con ramas dentro de un repositorio. Permite 
moverte entre ellas y modificar el proyecto sin hacer cambios dentro del main.
